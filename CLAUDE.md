### Decap CMS

`public/admin/` (index.html + config.yml) is served statically. Auth is the `github` backend via Netlify's hosted OAuth service (`base_url: https://api.netlify.com`) вАФ NOT Netlify Identity, which is deprecated and was removed from this setup. Requires a GitHub OAuth App (callback `https://api.netlify.com/auth/done`) installed under Netlify Site configuration вЖТ Access & security вЖТ OAuth. Editors log in with a GitHub account that has write access to the repo. CMS saves commit directly to `main`, which triggers a Netlify rebuild; there is no local CMS backend.

### Page shell pattern

Every page renders `Header` + `Footer` + `FloatingButtons` (WhatsApp/phone) + `AccessibilityWidget` itself вАФ there is no shared layout component. New pages must include all four and a `Helmet` block (react-helmet-async) with Hebrew title/description.

### Header navigation

`Header.tsx` nav items scroll to section ids on `/`; from other pages it navigates to `/#<id>`, and `Index.tsx` handles the hash-scroll in a `useEffect`. Keep the nav order: „™„Ч„Х„Ю„Щ „Ґ„Щ„°„Х„І before „Р„Х„У„Х„™ (explicit client requirement).

### Styling system

- Colors are HSL CSS variables in `src/index.css` (`--background`, `--accent` = gold, etc.) consumed via Tailwind tokens; the "legacy" tokens (`--gold`, `--navy`, `--cream`) are remapped to the same palette.
- Accessibility widget (`src/components/AccessibilityWidget.tsx`) works by setting `data-a11y-*` attributes on `<html>`; the corresponding CSS rules live in `src/index.css`. Any new styling must not break under `data-a11y-contrast="high"` or `data-a11y-font` scaling.
- `.font-tanakh` (Frank Ruhl Libre) is the biblical-style font used for the client's personal quotes (`QuoteStrip` in Index.tsx).
- CSS `@import` statements must stay above the `@tailwind` directives in `index.css` (vite:css warns otherwise).

### Language switcher

`src/components/LanguageSwitcher.tsx` uses the Google Translate cookie mechanism (`googtrans` cookie + injected script), not real i18n. Source language is `he`. Related CSS (banner hiding) is at the bottom of `index.css`. Elements that must never be machine-translated get the `notranslate` class.

### Images

Static images are plain files in `public/` (`/logo.jpeg`, `/shoshani-portrait.jpeg`, `/mitgashrim-logo.jpeg`). The `src/assets/*.asset.json` files are the old Lovable-hosted asset pointers вАФ they only resolve on Lovable hosting; don't add new ones.

### Contact form (Netlify Forms)

`src/components/sections/ContactForm.tsx` posts urlencoded data to `/` with `form-name=contact` (Netlify Forms). A hidden static mirror of the form lives in `index.html` so Netlify's build-time scanner registers it вАФ keep the field names (`name`, `phone`, `email`, `message`, `bot-field`) in sync between both files. Submissions only actually deliver when the site is hosted on Netlify.

### SEO invariants

Exactly one `<h1>` per page; every `<img>` must have a Hebrew `alt`. Static fallback title/description live in `index.html`; per-page values come from each page's `Helmet` block.
